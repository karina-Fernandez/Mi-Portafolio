SEMANA 2

Visual Studio Code y GitHub

Herramientas de desarrollo profesional, control de versiones, colaboración en proyectos y flujos de trabajo modernos en el desarrollo de software.

Temas Aprendidos

Visual Studio Code - Editor Profesional

Visual Studio Code es un editor de código fuente gratuito y de código abierto desarrollado por Microsoft. Ofrece soporte para múltiples lenguajes de programación, depuración integrada, control de versiones Git, terminal integrado y un ecosistema extenso de extensiones. Su interfaz intuitiva y características avanzadas como IntelliSense, refactoring automático y snippets lo convierten en una herramienta esencial para desarrolladores modernos.

GitHub - Plataforma de Colaboración

GitHub es una plataforma de desarrollo colaborativo basada en Git que permite a los desarrolladores almacenar, gestionar y colaborar en proyectos de código. Proporciona funcionalidades como repositorios remotos, issues tracking, pull requests, GitHub Actions para CI/CD, y GitHub Pages para hosting. Es fundamental para el trabajo en equipo, contribuciones open source y gestión de proyectos de software.

Control de Versiones con Git

Git es un sistema de control de versiones distribuido que rastrea cambios en archivos y coordina el trabajo entre múltiples desarrolladores. Permite crear ramas para desarrollo paralelo, fusionar cambios, revertir modificaciones y mantener un historial completo del proyecto. Los conceptos clave incluyen commits, branches, merges, y workflows como GitFlow para gestión eficiente de releases.

Flujos de Trabajo Colaborativos

Los flujos de trabajo colaborativos definen cómo los equipos organizan y gestionan el desarrollo de software. Incluyen prácticas como feature branches, pull requests para revisión de código, integración continua, y deployment automático. Estos procesos aseguran calidad del código, facilitan la colaboración y minimizan conflictos en proyectos con múltiples contribuidores.

Ejercicios de Laboratorio - Resultados

Ejercicio 01

Configuración avanzada de VSC

Objetivo: Personalizar el entorno de desarrollo y instalar extensiones esenciales.

Configuré VSC con extensiones fundamentales como Prettier para formateo automático, ESLint para análisis de código, Live Server para desarrollo web, y GitLens para visualización avanzada de Git. Personalicé temas, configuré atajos de teclado personalizados y establecí configuraciones de workspace para diferentes tipos de proyectos.

Ejercicio 02

Creación y gestión de repositorios

Objetivo: Crear repositorios en GitHub y conectarlos con proyectos locales.

Creé repositorios tanto locales como remotos en GitHub, establecí conexiones SSH para autenticación segura, configuré archivos .gitignore apropiados para diferentes tipos de proyectos, y practiqué la sincronización entre repositorios locales y remotos usando comandos como clone, fetch, pull y push.

Ejercicio 03

Branching y merging strategies

Objetivo: Implementar estrategias de ramificación y fusión de código.

Practiqué la creación de ramas para features específicos, implementé el flujo GitFlow con ramas develop, feature, release y hotfix. Aprendí a resolver conflictos de merge, usar rebase para mantener un historial limpio, y aplicar estrategias de merge como fast-forward y merge commits según el contexto del proyecto.

Ejercicio 04

Pull requests y code review

Objetivo: Implementar procesos de revisión de código colaborativo.

Creé pull requests detallados con descripciones claras, implementé templates para PRs, practiqué la revisión de código proporcionando feedback constructivo, configuré branch protection rules para requerir revisiones antes de merge, y utilicé GitHub's review tools para sugerir cambios específicos en líneas de código.

Ejercicio 05

GitHub Actions y automatización

Objetivo: Configurar pipelines de CI/CD básicos con GitHub Actions.

Configuré workflows básicos de GitHub Actions para automatizar testing, linting y deployment. Creé actions para ejecutar tests automáticamente en cada push, configuré deployment automático a GitHub Pages, implementé notificaciones de estado de builds, y exploré el marketplace de actions para integrar herramientas externas como Slack y Discord.
